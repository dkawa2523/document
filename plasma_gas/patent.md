1. 1ページ要約

発明の要旨

本技術は、低圧プラズマのサロゲートモデルを構築する際に、従来のようにガス種ごと・混合比ごとに別データセットと別モデルを都度作り込む方式ではなく、
断面積データセット管理 → swarm シミュレーション → 輸送係数・反応係数表 → ガス効果の潜在表現化 → 二段AI推論
という層分離構成に置き換えるものである。

中核は、レシピ名やガス名を直接AIに入れるのではなく、swarm 計算で得られる移動度、拡散係数、反応係数、平均電子エネルギー等の“物理表現”を特徴量化し、ガス効果を潜在表現として学習させる点にある。これにより、想定ガスの追加や混合比変更時でも、必要な追加シミュレーション数と再学習コストを抑えつつ、空間場の推定精度と適用可否判定を両立できる。

新規性の核3点

項目	内容
核1	断面積レジストリを起点に、xs_set_id、完全性、理論前提、温度適用範囲、品質状態を付与し、再現可能なデータ生成パイプラインを構成する点
核2	ガス名や one-hot ではなく、swarm 由来の輸送係数曲線・反応係数曲線を潜在表現化してガス効果を表現し、第1AIで係数を、第2AIで空間場を扱う二段構成を採る点
核3	予測値だけでなく、信頼度指標を、モデル不確かさ・物理残差・潜在空間距離・断面積品質情報から算出し、適用可否を明示する点

期待効果の定量

以下の数値は**導入効果の想定レンジ【推定】**であり、装置構成、教師シミュレーションの種類、対象ガス系に依存する。

効果指標	想定効果
新規ガス組成追加時の高コストシミュレーション件数	40〜90%削減【推定】
ガス変更時のデータ準備・再学習工数	30〜70%短縮【推定】
フル物理シミュレーション対比の推論速度	10³〜10⁶倍高速化【推定】
out-of-distribution 条件の検知	信頼度指標により自動警告可能
データ再現性・監査性	xs_set_id、run_id、manifest により追跡性向上

2. 従来技術（背景）と先行技術カテゴリ

半導体製造に用いられる低圧プラズマでは、ガス種、混合比、圧力、印加条件が変わると、電子輸送・反応係数が大きく変化し、結果として電子密度、電位分布、ソース項、イオンフラックス分布などの場が変わる。そのため、従来は物理シミュレーションに強く依存してきた。

一方で、近年はサロゲートモデルが検討されているが、ガス種ごとにデータセットを分ける、またはレシピ条件をそのままAI入力する方式が多く、ガスの追加や断面積セット変更に弱い場合がある。

想定される先行技術カテゴリ

カテゴリ	概要	長所	主な限界
C1 物理シミュレーション直実行	swarm、fluid、PIC等を条件ごとに実行	物理整合性が高い	計算時間が長く、探索に不向き
C2 ガス種ごとの個別サロゲート	ガス系ごとに別学習	単一系では作りやすい	新ガス導入時に作り直しが必要
C3 レシピ条件直入力型AI	ガス名、流量、圧力等をそのまま入力	実装しやすい	物理的意味が薄く、外挿に弱い
C4 物理制約付きAI	PDE残差等をLossへ追加	整合性向上	入力表現が弱いと汎化に限界
C5 DB管理＋手動解析	断面積や係数表を人手で管理	再現性は一定	AI連携や自動拡張が弱い

従来課題は、特に以下の条件で顕在化する。

指標	問題点

	•	実運用では、予測値そのものに加えて、どこまで信用できるかの説明が必要になる

4. 提案手法

4-1. システム構成（文章）

本提案は、少なくとも以下の機能部から構成される。
	1.	断面積レジストリ部
断面積データセットを xs_set_id により識別し、種別、版、完全性、理論前提、品質状態を管理する。
	2.	実行計画部
ガス組成、圧力、ガス温度、E/N または平均電子エネルギー、必要に応じて波形条件等をサンプリングし、swarm 計算の manifest を生成する。
	3.	swarm 実行部
保有済みの swarm シミュレーションコードを用いて、輸送係数、反応係数、平均電子エネルギー等を取得する。
	4.	係数表生成部
raw 出力を、共通グリッドに再標本化し、E/N 軸テーブル、必要に応じて平均電子エネルギー軸テーブルへ整形する。
	5.	特徴量生成部
反応効果特徴量、ガス効果特徴量、信頼度補助特徴量を生成する。
	6.	第1AIモデル部
ガス組成、潜在表現、運転条件から輸送係数・反応係数を推定する。
	7.	係数マップ生成部
第1AIの出力を空間入力用の係数マップへ変換する。
	8.	第2AIモデル部
幾何・境界条件・係数マップから電子密度、イオン密度、電位等の空間場を推定する。
	9.	信頼度算出部
不確かさ、物理残差、潜在距離、断面積品質情報に基づき信頼度を算出する。

4-3. 反応効果とガス効果の特徴量化仕様

本提案では、“レシピ名”ではなく“物理表現”を特徴量にする点が重要である。

4-3-1. ガス効果特徴量

a. 組成特徴量
ガス組成 x=(x_1,\dots,x_n) は総和1の制約を持つため、そのままよりも log-ratio 変換が有利である。

z_{comp}=ilr(x) \quad \text{または} \quad clr(x)

これにより、組成データをユークリッド空間で扱いやすくする。

b. 輸送潜在表現
共通グリッド上に再標本化した係数曲線群を入力し、エンコーダで潜在表現へ圧縮する。

z_{gas}=Enc\Big(
[\log \mu N,\log D_LN,\log D_TN,\log k_{ion},\log k_{att},\log k_{exc,grp},\bar{\epsilon}]
\Big)

この z_gas が、ガス種・混合比・断面積差分を含むガス効果の表現となる。

c. 断面積潜在表現【任意】
raw cross section 曲線群を process token としてエンコードし、z_xs を生成してもよい。
これは新規ガス種追加時の拡張性に有効である【推定】。

4-3-2. 反応効果特徴量

反応効果は、反応レートそのものだけでなく、反応間競合やしきい値感度を含めて表現する。

f_{rxn}=
[
\log k_{ion},
\log k_{att},
\log k_{exc,grp},
\log k_{diss,grp},
\rho_{ion/att},
s_{thr},
p_{part}
]

ここで、
	•	\rho_{ion/att}：電離と付着の競合指標
	•	s_{thr}：しきい値近傍での勾配や感度
	•	p_{part}：elastic / excitation / ionization 等へのエネルギー配分

とする。

4-3-3. 信頼度補助特徴量

f_{conf}=
[
u_{model},
r_{phys},
d_{latent},
q_{xs}
]

記号	意味
u_{model}	アンサンブル分散等のモデル不確かさ
r_{phys}	物理残差。例：Poisson残差、連続式残差
d_{latent}	学習済み潜在空間からの距離
q_{xs}	断面積セットの品質指標

4-3-4. 前処理仕様

項目	前処理
係数値	floor 付き log 変換
組成	ilr / clr 変換
曲線	共通 E/N グリッドまたは平均電子エネルギーグリッドに再標本化
反応	物理意味に基づくグルーピング
入力値	無次元化または正規化

4-4. AIモデル仕様

4-4-1. 全体構成

本提案は、二段AI構成を採る。
	1.	第1AIモデル

発明上の芯	係数マップ条件付けにより、ガス変更の影響を空間場へ一貫伝播すること

4-4-4. Loss 仕様

第1AI用 Loss の一例
L_1=
\lambda_1 L_{\log}
+\lambda_2 L_{rel}
+\lambda_3 L_{smooth}
+\lambda_4 L_{latent}
	•	L_{\log}：log 空間の Huber / L1 / L2
	•	L_{rel}：相対誤差
	•	L_{smooth}：曲線の一次・二次勾配整合
	•	L_{latent}：潜在表現の再構成または整合

第2AI用 Loss の一例
L_2=
\alpha_1 L_{field}
+\alpha_2 L_{\nabla}
+\alpha_3 L_{BC}
+\alpha_4 L_{PDE}
+\alpha_5 L_{int}
+\alpha_6 L_{roll}

項	意味
L_{field}	密度・電位のデータ誤差
L_{\nabla}	勾配整合。シース境界や急峻部を保持
L_{BC}	境界条件誤差
L_{PDE}	Poisson、連続式等の物理残差
L_{int}	全電流、全パワー、総フラックス等の積分量誤差
L_{roll}	時系列・周期整合【任意】

4-4-5. 信頼度算出

推定結果に対し、信頼度を併記する。

Score_{conf}
=
1-
Norm(
w_1 u_{model}
+w_2 r_{phys}
+w_3 d_{latent}
+w_4 (1-q_{xs})
)

この信頼度は、例えば
	•	高信頼：結果をそのまま利用
	•	中信頼：要監査
	•	低信頼：高忠実度シミュレーションへフォールバック
のように運用できる【推定】。

4-4-6. 学習手順の一例
	1.	断面積レジストリ整備
	2.	swarm 実行と係数表生成
	3.	潜在表現エンコーダ学習
	4.	第1AI学習
	5.	第2AI学習
	6.	信頼度校正
	7.	誤差・不確かさが高い条件への追加計算

7. 新規性・進歩性

以下は発明の差別化ポイント整理であり、最終的な新規性・進歩性判断には先行特許調査・文献調査が必要である【要確認】。

新規性候補

観点	従来想定	本提案との差分
入力設計	ガス名、流量、圧力を直接入力	swarm由来係数曲線を潜在表現化して入力
データ生成	ガス系ごとに個別収集	断面積レジストリ起点で再利用可能な係数表を生成
モデル構造	単一AIにまとめる	第1AIで係数、第2AIで空間場の二段構成
適用判定	予測値のみ	信頼度指標を同時出力
更新方式	新ガスで大規模再学習	追加シミュレーションを選択的に実施

進歩性の論点
	1.	断面積データ管理とAI学習を一体設計している点
単なるDB管理でも単なるAIでもなく、出力される係数表の品質と再現性が、学習の安定性・運用信頼性に結び付く構成である。
	2.	ガス効果の表現方法が、単なる組成値ではなく“物理曲線の潜在表現”である点
これにより、新規組成や近傍組成への転移が自然になる。
	3.	係数予測と空間場予測を分離し、係数マップで結ぶ点
物理的意味が明確で、更新影響を局所化できる。
	4.	信頼度を断面積品質まで含めて算出する点
AI不確かさだけではなく、入力データ源の品質を運用に反映する。

有用性

	•	新規ガス導入時の検討負荷軽減

	4.	ガス効果潜在表現を学習
	5.	第1AIで係数を予測
	6.	第2AIで電子密度・電位・ソース項を予測
	7.	信頼度を付与して結果提示

効果
	•	新規組成候補のスクリーニング高速化【推定】
