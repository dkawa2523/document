1. 1ページ要約

1-1. 発明の要旨

本技術は、半導体ウェハ上に観測された付着粒子を、粒子座標（二次元）と粒子サイズ（一次元）を直接入力として扱い、ウェハ全体を画像へラスタ化せずに、粒子単位で原因パターンラベルを推定する技術である。
特に、実運用で重要となる 複数パターン混合ウェハ に対し、単一パターン教師データしか十分に存在しない場合でも、複数の単一パターンウェハを合成して粒子単位ラベル付き混合教師データを自動生成し、これを用いて学習可能とする。

推論時には、ウェハを
	•	粒子ノード
	•	半径方向アンカーノード
	•	角度方向アンカーノード
	•	ウェハ全体ノード

からなる固定マルチグラフとして構成し、各関係に対して relation-specific NNConv を適用する。
これにより、粒子近傍の局所幾何だけでなく、リング状・セクタ状・エッジ起点・放射状・ホットスポット状など、ウェハ全体で意味を持つ分布文脈を同時に利用できる。

さらに、粒子ラベルだけでなく、
	•	近傍粒子どうしが同一原因成分に属するか
	•	ウェハ全体にどのパターン群が存在するか
	•	予測の不確かさがどれほど高いか

を併せて推定し、自動アクション／要レビュー分岐／再学習へ接続する点に特徴がある。

1-2. 新規性の核 3 点
	1.	単一パターン教師から混合ウェハ教師を自動生成する学習データ生成方法
既存の単一ラベルウェハを複数選択し、粒子を thinning・回転・微小摂動・背景付与して合成しつつ、各粒子の元パターンラベルと component_id を保持する点。
	2.	粒子・半径アンカー・角度アンカー・global を併用した固定マルチグラフ表現
粒子間の kNN/radius 関係に加え、半径帯と角度帯のアンカーを導入し、局所構造とウェハ全体分布文脈を同じ GNN に流す点。
	3.	粒子ラベル・同一成分判定・グラフ多ラベル・不確かさ分岐を一体化
単なる分類ではなく、要因切り分け用途を念頭に、粒子単位ラベル、成分分解支援、混合検出、不確かさに応じたレビュー制御を一体化した点。

3. 従来の課題（発生条件、現行対策の限界、評価指標、制約）

3-1. 発生条件

本問題は、以下の条件で特に難しくなる。

条件	内容
混合パターン	1枚のウェハ上に、リング、線状、hotspot、エッジ起点など複数原因が混在
少粒子数	粒子数が少ないため、統計的に十分な形状が見えにくい
粒子単位出力要求	ウェハ全体の1ラベルでは足りず、粒子ごとの原因ラベルが必要
教師不足	実運用では、混合ウェハの粒子単位ラベルは高コストでほぼ得られない
製造判断接続	推定結果をレビュー優先度、装置点検、工程条件調整に結び付けたい

3-2. 現行対策の限界
	1.	画像化ベースの限界
粒子数が少ない場合、画像化すると多くが空白となり、局所密度や線形成分が不安定化する。さらに粒子の mm 座標をピクセルへ量子化するため、空間精度が損なわれる。
	2.	ルールベースの限界
単純な半径集中や角度集中では、混合ケースで説明が破綻しやすい。例えば edge-biased と crescent-edge、straight-line と radial-lines の切り分けは、単一統計量では困難である。
	3.	サンプル分類の限界
1ウェハ1クラス分類では、複数原因が共存した場合に「どの粒子がどの原因か」を返せない。
	4.	信頼度不在の限界
少粒子ケースでは、本質的に判別不能な場合がある。にもかかわらず常に top-1 を断定すると、製造現場で誤アクションにつながる。

4. 提案手法

4-1. システム構成（文章）

本技術の一実施形態では、ウェハ検査装置またはその後段解析装置から取得された粒子テーブルを入力し、前処理部が各粒子の座標・粒径を正規化するとともに、半径方向・角度方向・局所密度・局所線形性等の特徴量を算出する。

学習時には、単一パターン教師ウェハ群から複数のウェハを選択し、それぞれの粒子集合を thinning、回転、微小位置摂動、粒径摂動、背景成分付与等により合成して、粒子単位ラベル付き混合教師グラフを生成する。各粒子は元の component_id と source label を保持する。

次いで、グラフ生成部が、各ウェハについて、
	•	粒子ノード
	•	半径アンカーノード
	•	角度アンカーノード
	•	globalノード

を含む異種グラフを生成し、
	•	k近傍粒子間エッジ
	•	半径閾値内粒子間エッジ
	•	粒子−半径アンカーエッジ
	•	粒子−角度アンカーエッジ
	•	粒子−globalエッジ

を構成する。

推論部は、各関係に対して独立に重み付けされる graph convolution を実施し、粒子ごとのパターンラベル、近傍粒子が同一原因成分に属するか否か、及びウェハ全体に存在するパターン集合を出力する。
その後、不確かさ算出部が粒子単位およびウェハ単位の信頼度を算出し、閾値以上の場合は自動アクション候補を提示し、閾値未満の場合はレビューキューへ送る。レビュー結果は、任意で再学習用データベースへ蓄積される。

4-3-3. 教師生成方法

単一パターン教師のみ存在する場合、以下で mixed teacher を作る。
	1.	2 個又は 3 個の source class を選ぶ
	2.	各 class から 1 ウェハ選択
	3.	Dirichlet 重みで採用粒子数を決める
	4.	各成分を回転、微小座標 jitter、粒径 jitter する
	5.	必要に応じ一様背景成分を追加する
	6.	粒子を連結し、component_id と node_label を保持する

一実装例では、
	•	混合成分数：2 または 3
	•	混合後粒子数：40〜220
	•	座標 jitter：1.0 mm
	•	粒径 jitter：対数正規的微小摂動
	•	一様背景追加確率：0.35

とする。

4-4. AIモデル仕様

4-4-1. グラフ定義

1ウェハを 1 個の異種グラフ
G=(V_p, V_r, V_a, V_g, E_{knn}, E_{rad}, E_{pr}, E_{rp}, E_{pa}, E_{ap}, E_{pg}, E_{gp})
として定義する。
	•	V_p：粒子ノード
	•	V_r：半径アンカーノード
	•	V_a：角度アンカーノード
	•	V_g：globalノード

一実装例では、
	•	kNN：k=8
	•	radius scales：0.06R, 0.12R
（ウェハ半径150 mm前提では約9 mm, 18 mm）
	•	radial bins：8
	•	angular bins：12
	•	anchor neighbors：1

とする。

4-4-2. relation-specific NNConv

各関係 r ごとに、エッジ属性 e_{ij}^{(r)} から重み行列を生成し、メッセージ伝播する。

m_{ij}^{(r,\ell)} = W_r(e_{ij}^{(r)}) \, h_j^{(\ell)}

\tilde{h}_i^{(\ell+1)} = \sum_{r} \sum_{j \in \mathcal{N}_r(i)} m_{ij}^{(r,\ell)}

h_i^{(\ell+1)} = \mathrm{LayerNorm}\left(h_i^{(\ell)} + \mathrm{GELU}(\tilde{h}_i^{(\ell+1)})\right)

これを複数層スタックし、粒子局所特徴とアンカー文脈を融合する。
一実装例では hidden dim=64、3層、dropout=0.1 とする。

4-4-3. 出力ヘッド

ヘッド	出力	目的
Node head	粒子ごとのクラス logits	粒子単位ラベル推定
Edge head	kNN辺ごとの same-component logits	原因成分分離支援
Graph head	ウェハ全体の multi-label logits	混合パターン存在検出

4-4-4. 損失関数

L = \lambda_n L_{node} + \lambda_e L_{edge} + \lambda_g L_{graph}

一実装例では、
	•	\lambda_n = 1.0
	•	\lambda_e = 0.4
	•	\lambda_g = 0.3

とする。

各損失の例：
	•	L_{node}：Cross Entropy
	•	L_{edge}：Binary Cross Entropy
	•	L_{graph}：Binary Cross Entropy with Logits

label smoothing を 0.02 としてもよい。

4-4-5. 学習カリキュラム
	1.	単一パターン事前学習
元ウェハをそのままグラフ化し、粒子ラベル=ウェハラベルとして学習する。
	2.	混合ウェハ微調整
on-the-fly 生成した混合グラフで、node/edge/graph の多タスク学習を行う。

これにより、初期の幾何表現を安定化しつつ、最終目的である mixed wafer へ適応させる。

4-4-6. 不確かさ/信頼度

本技術では、少粒子・曖昧ケースに対し、不確かさを出力することを重視する。

粒子単位の不確かさの例：
u_i = 1 - \max_c p(c|i)
又はエントロピー
u_i = -\sum_c p(c|i)\log p(c|i)

ウェハ単位不確かさの例：
	•	graph head の最大確率
	•	multi-label entropy
	•	温度スケーリング後の calibration 指標

4-4-7. 成分分離

edge head が出す same-component 確率に閾値 \tau_e を設け、閾値以上の辺で粒子を連結し、Union-Find または connected components により component を推定する。
これにより、「どの粒子群が同じ原因由来か」を抽出できる。

4-4-8. 一実装例のアルゴリズム要約

学習フェーズ
	•	S1: 単一パターン粒子データ読込
	•	S2: train/val/test 分割
	•	S3: 単一ウェハグラフで事前学習
	•	S4: 単一ウェハを複数選択し mixed graph を生成
	•	S5: 固定マルチグラフ構築
	•	S6: node/edge/graph 多タスク学習
	•	S7: 最良モデル保存

推論フェーズ
	•	S8: 粒子テーブル入力
	•	S9: グラフ構築と推論
	•	S10: 粒子ラベル、不確かさ、component 候補を出力
	•	S11: 閾値分岐で自動提案またはレビュー送出

7. 新規性・進歩性

7-1. 新規性の候補

候補1

単一パターンウェハ群から、粒子単位ラベルと component_id を保持した混合ウェハ教師を生成すること。
これは単なるデータ拡張ではなく、最終タスクである「混合ウェハ入力 → 粒子単位出力」へ教師構造を合わせる点に特徴がある。

候補2

粒子・半径アンカー・角度アンカー・global を併用した固定マルチグラフ表現。
一般的 point cloud/GNN は点間関係中心であるが、本技術はウェハ特有の極座標構造をアンカーとして陽に入れる。

候補3

粒子分類、成分分離、混合検出、不確かさ分岐を製造アクションまで結ぶ一体設計。
単なる AI 精度向上ではなく、要因切り分けという製造課題に合わせた構成である。

7-2. 進歩性の候補
	1.	画像CNNへの自明置換ではない点
本技術は画像化をやめ、粒子座標と粒径を直接用い、極座標アンカーで全体文脈を扱う。これは単純な入力形式変更ではなく、問題構造に合わせた表現設計である。
	2.	汎用GNNの単純適用ではない点
kNN だけでなく、半径閾値辺、半径アンカー、角度アンカー、global を固定的に設け、さらに relation-specific NNConv を使う構成は、ウェハ付着マップの幾何と要因切り分け目的に合わせている。
	3.	学習データ生成方法まで含む点
混合タスクのための教師不足を、合成・component保持・多タスク教師化で解決する。これは推論器単体の発明より実務上の価値が高い。

7-3. 有用性
	•	粒子ごとのラベル出力により、レビュー工数や再解析工数を削減しうる
	•	少粒子ケースでも「断定不可」を扱える
	•	pattern family ごとに装置部位・工程起点の仮説整理に使える
	•	将来的に recipe / tool / chamber 条件との相関解析へ拡張しやすい

8. 実施例

実施例1（代表ケース）：添付データを用いた混合ウェハ粒子ラベル学習

条件

項目	内容
入力データ	添付 particles.csv, samples.csv
元データ規模	15,000ウェハ、約165万粒子、15クラス
粒子数	20〜200粒子/ウェハ
ウェハ半径	150 mm
学習分割	StratifiedGroupKFold による概ね 80/10/10
graph 構成	particle + rbin(8) + abin(12) + global
local edges	kNN(k=8) + radius(0.06R, 0.12R)
学習	単一パターン事前学習 → mixed fine-tune
mixed 条件	2〜3成分、40〜220粒子、回転あり、1.0mm jitter

手順
	1.	単一パターンウェハを読み込む
	2.	粒子特徴量とアンカー特徴量を算出する
	3.	単一ウェハをグラフ化して事前学習する
	4.	2〜3枚の source wafer を選び、粒子を間引きして混合グラフを生成する
	5.	各粒子に source class と component_id を保持する
	6.	fixed multi-graph + NNConv で多タスク学習する
	7.	推論時に粒子ラベル、same-component、graph multilabel、不確かさを出力する

結果

実測精度は未取得であり、以下は本実施例で得られる出力内容である。
	•	各粒子について 15 クラスの事後確率
	•	各局所辺について same-component 確率
	•	ウェハ全体に存在するクラス集合の確率
	•	各粒子群の component 推定
	•	高信頼/低信頼の分岐フラグ

技術的意義

本実施例では、追加の mixed 粒子手付けラベルを作成せずに、混合タスクの学習が可能となる。
また、粒子数が 40〜220 程度の疎点群であっても、アンカー分岐によりウェハ全体文脈を保ったまま推論できる。

実施例2（変形例）：低粒子数ウェハの要因切り分け支援

条件

項目	内容
対象	少粒子ウェハ（例：40粒子未満又は40〜60粒子程度）
目的	自動断定ではなく、原因候補の絞込みとレビュー優先度付け
推論出力	粒子ラベル、family-level ラベル、component 候補、不確かさ
分岐	高信頼は自動提案、低信頼は review queue

手順
	1.	推論対象ウェハの粒子グラフを生成する
	2.	node/edge/graph 出力を計算する
	3.	粒子単位最大確率、グラフ単位 entropy、component 一貫性を確認する
	4.	所定閾値を下回る場合、fine label ではなく family-level 候補のみ提示する
	5.	レビュー結果を追加入力し、任意で再学習する

結果

要確認ではあるが、本変形例では、少粒子で判別が難しい場合にも「不明」や「family候補」に落とし込めるため、誤自動化を抑えやすい。
例えば、
	•	edge 起点系
	•	hotspot 系
	•	line/scratch 系

のような family 候補を提示し、人が工程要因を点検しやすくなる。

技術的意義

実運用では「必ず正答ラベルを出す」よりも、「高信頼なら自動提案、低信頼ならレビュー」が有用である。
本変形例は、まさにその製造運用へ接続しやすい。

11-6. クレーム設計メモ（広いクレーム/狭いクレーム、回避設計ポイント）

広いクレームで押さえたい軸
	1.	粒子座標 + 粒子属性を直接使うこと
	2.	ウェハ文脈ノードを持つグラフ表現
	3.	関係別メッセージ伝播
	4.	粒子単位ラベル + 信頼度分岐
	5.	混合教師生成を含む学習方法

狭いクレームで補強したい軸
	1.	半径アンカー + 角度アンカー + global の3種
	2.	kNN + radius + anchor edges の併用
	3.	same-component 補助タスク
	4.	単一教師からの mixed teacher 自動生成
	5.	不確かさに応じたレビュー/再学習

回避設計ポイント
	•	画像ラスタ化 + Transformer/CNN による実装
	•	Set Transformer など非グラフ型 set model
	•	anchor を使わず global feature だけを注入する構成
	•	NNConv ではなく他の relation-aware conv を使う構成

回避設計への備え
	•	独立請求項では NNConv に限定しすぎず、「関係ごとに異なる重み付けを行うグラフニューラルネットワーク」と広く表現する
	•	「半径アンカー」「角度アンカー」は従属請求項で具体化し、独立請求項では「ウェハ全体文脈を表す補助ノード群」としてもよい
	•	「粒子サイズ」は独立請求項では「少なくとも1つの粒子属性」と広げる余地がある
